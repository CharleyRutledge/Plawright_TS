# Example Test Plan

## Application Overview

This is an example test plan that demonstrates the structure expected by the Playwright generator agent.

**Application**: Playwright Documentation Website  
**Base URL**: https://playwright.dev  
**Purpose**: Verify core navigation and documentation access

## Test Scenarios

### 1. Homepage Navigation

**Seed:** `tests/seed.spec.ts`

#### 1.1 Verify Homepage Loads

**Steps:**
1. Navigate to the homepage
2. Verify page title contains "Playwright"

**Expected Results:**
- Page title is visible and contains "Playwright"
- Main heading is displayed
- Navigation menu is visible

#### 1.2 Navigate to Documentation

**Steps:**
1. Navigate to the homepage
2. Click on the "Get started" link
3. Verify redirection to documentation page

**Expected Results:**
- URL changes to documentation section
- "Installation" heading is visible
- Installation instructions are displayed

### 2. Search Functionality

**Seed:** `tests/seed.spec.ts`

#### 2.1 Open Search Dialog

**Steps:**
1. Navigate to the homepage
2. Click on the Search button
3. Verify search dialog opens

**Expected Results:**
- Search input field becomes visible
- Search input field is focused
- User can type in search field

#### 2.2 Perform Search

**Steps:**
1. Navigate to the homepage
2. Open search dialog
3. Type "test" in search field
4. Press Enter or click search result

**Expected Results:**
- Search results are displayed
- Results are relevant to search query
- User can navigate to selected result

### 3. API Documentation Access

**Seed:** `tests/seed.spec.ts`

#### 3.1 Navigate to API Documentation

**Steps:**
1. Navigate to the homepage
2. Click on "API" link in navigation
3. Verify API documentation loads

**Expected Results:**
- URL changes to API section
- API reference content is visible
- Navigation sidebar shows API categories

## Notes

This test plan is generated by the **planner** agent and consumed by the **generator** agent to create executable Playwright tests. If tests fail, the **healer** agent will automatically repair them.
